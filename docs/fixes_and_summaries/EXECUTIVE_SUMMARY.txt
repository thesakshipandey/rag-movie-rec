================================================================================
                            KEY FEATURES
================================================================================

ROUTER ARCHITECTURE:
- Context Feature Encoder (handles emotion, categorical, numerical features)
- MLP Router (3 layers with LayerNorm and Dropout)
- Softmax output over 4 experts (alpha, beta, gamma, delta)
- Learnable temperature for calibration

TRAINING:
- ListMLE loss (Plackett-Luce likelihood)
- 70/15/15 train/val/test split with random shuffle
- Entropy regularization to prevent collapse
- AdamW optimizer with cosine annealing
- Automatic checkpointing (saves best validation loss)

EVALUATION:
- nDCG@10, MRR, Hit@10 metrics
- Single expert baselines
- Uniform and oracle mixtures
- Comprehensive visualizations
- Per-prompt analysis

Z-SCORE NORMALIZATION:
- Applied per expert per prompt before combination
- Ensures fair comparison across different score scales
- Formula: z = (score - mean) / std

================================================================================
                        QUICK START COMMANDS
================================================================================

COMPLETE PIPELINE (ONE COMMAND):
    bash projects/rag-movie-rec/run_listwise_pipeline.sh

OR STEP BY STEP:

1. GENERATE EXPERT SCORES (~15-30 min):
    python -m src.router.generate_expert_scores \
        --data_dir projects/Data \
        --indices_dir projects/rag-movie-rec/artifacts/indices \
        --out projects/rag-movie-rec/artifacts/router/listwise_expert_scores.parquet

2. TRAIN ROUTER (~10-20 min):
    python -m src.cli.train_router_listwise \
        --expert_scores projects/rag-movie-rec/artifacts/router/listwise_expert_scores.parquet \
        --prompts_path projects/Data/prompts.json \
        --out projects/rag-movie-rec/artifacts/router/router_listwise.pt \
        --epochs 50

3. EVALUATE SINGLE EXPERTS (~5 min):
    python -m src.evaluations.eval_single_experts \
        --expert_scores projects/rag-movie-rec/artifacts/router/listwise_expert_scores.parquet \
        --prompts_path projects/Data/prompts.json \
        --out projects/rag-movie-rec/artifacts/evaluation_results/listwise/single_expert_metrics.json

4. EVALUATE ROUTER (~5 min):
    python -m src.evaluations.eval_router_listwise \
        --expert_scores projects/rag-movie-rec/artifacts/router/listwise_expert_scores.parquet \
        --prompts_path projects/Data/prompts.json \
        --router_checkpoint projects/rag-movie-rec/artifacts/router/router_listwise.pt \
        --out projects/rag-movie-rec/artifacts/evaluation_results/listwise/router_metrics.json

5. COMPARE ALL METHODS (~1 min):
    python -m src.evaluations.compare_methods \
        --single_expert_metrics projects/rag-movie-rec/artifacts/evaluation_results/listwise/single_expert_metrics.json \
        --router_metrics projects/rag-movie-rec/artifacts/evaluation_results/listwise/router_metrics.json \
        --out_dir projects/rag-movie-rec/artifacts/evaluation_results/listwise/comparison

================================================================================
                           TROUBLESHOOTING
================================================================================

Issue: CUDA out of memory
→ Use --device cpu or reduce --batch_size 16

Issue: Router collapses to single expert
→ Increase --entropy_weight 0.01 or lower --entropy_target 1.0

Issue: All expert scores are zero
→ Check that indices are properly loaded (FAISS, BM25, etc.)

Issue: Poor nDCG scores
→ Verify ground truth scores format (should be 1.0, 0.95, 0.93, ...)

For more troubleshooting, see LISTWISE_ROUTER_README.md section on
"Troubleshooting"

================================================================================
                          TECHNICAL DETAILS
================================================================================

DATA FORMAT:
- Input: merged_all.json (listwise rankings per prompt)
- Input: prompts.json (prompt metadata with mix_weights)
- Output: Parquet with expert z-scores + ground truth

TRAIN/VAL/TEST SPLIT:
- 70% train, 15% validation, 15% test
- Random shuffle (data is category-sorted)
- GroupShuffleSplit ensures no prompt leakage

LOSS FUNCTIONS:
- ListMLE (primary): Plackett-Luce likelihood
- ListNet (alternative): Cross-entropy on top-1 distributions
- ApproxNDCG (alternative): Differentiable nDCG approximation

METRICS:
- nDCG@K: Normalized Discounted Cumulative Gain
- MRR: Mean Reciprocal Rank
- Hit@K: Fraction of relevant items in top-K
